# 🎬 My Pocket Movie

> **“내가 찜한 영화, 나만의 컬렉션으로 정리하는 영화 커뮤니티 플랫폼”**  
> 팀 프로젝트에서 PM 역할을 수행하며 **기획 → 설계 → 실행 → 피드백 개선** 전 과정을 주도했습니다.  

---

## 📌 프로젝트 개요

**My Pocket Movie**는 사용자가 TMDB API를 통해 영화를 검색하고,  
리뷰·평점·댓글·좋아요 등 소셜 기능을 통해 **나만의 영화 컬렉션**을 관리할 수 있는 커뮤니티 플랫폼입니다.  

- **프로젝트 기간**: 2025.05 ~ 2025.07 (6주)  
- **팀 인원**: 4명 (백엔드 2, 프론트엔드 1, PM/기획 1)  
- **성과**: MVP를 6주 내 기획·개발·배포, 사용자 피드백 기반 기능 개선 수행  

---

## 🚀 주요 기여 

- **요구사항 정의 & 기능 기획**  
  - 사용자 시나리오 설계(검색·찜하기·리뷰 작성)  
  - RICE Framework로 기능 우선순위 결정 → MVP 범위 확정  

- **프로젝트 일정 & 협업 관리**  
  - GitHub Issues + Kanban 보드로 태스크 관리  
  - 스프린트 단위 점검·리스크 대응 회의 주도  

- **API & 데이터 설계 조율**  
  - 외부 영화 API(Open API) 연동 방식 검토  
  - 회원·영화·리뷰 테이블 스키마 설계 및 명세 문서화  

- **피드백 기반 개선**  
  - 사용자 테스트 진행 → 리뷰 UI/검색 성능 개선  
  - QA 시나리오 정의 및 기능 안정성 확보  

---

## 📊 프로젝트 성과

- **기한 내 MVP 배포 성공** (6주)  
- 사용자 피드백 기반 UI/UX 개선 및 검색 정확도 향상  
- GitHub 기반 협업 프로세스 최적화 → 팀 생산성 개선  
- 데이터 모델링 & API 문서화를 통해 **개발-기획 커뮤니케이션 비용 절감**

## 🛠 기술 스택

| 구분 | 기술 |
|------|------|

## 📁 프로젝트 아키텍처

- Controller → Service → Repository → Entity/DTO 계층형 구조  
- RESTful API 기반 설계 & Swagger 문서 자동화  
- GitHub Actions + CodeDeploy 통한 CI/CD 자동화  
- Parameter Store로 환경 변수 관리
