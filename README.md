# 🎬 My Pocket Movie — PM-Centric README

> **영화 애호가 전용 커뮤니티 MVP**  
> **PM 책임 범위:** 문제정의 · 목표/KPI 수립 · 범위/우선순위 · 일정/리스크 · 협업 거버넌스 · 품질/릴리스 · 학습/회고

---

## 0) Executive Summary (One-liner)
**“6주 내 영화 전용 커뮤니티 MVP를 기획·개발·배포하고, 사용자 피드백 기반으로 검색/리뷰 UX를 개선하여 핵심 KPI 초기 가설을 검증했다.”**

---

## 1) 문제 정의 (Why)
- **시장 공백:** 범용 SNS(인스타/페북/유튜브)는 존재하나 **영화 중심 소통/컬렉션 전용 플랫폼 부재**  
- **사용자 과제:** 감상 기록의 분산, 개인 컬렉션 구축/공유의 불편, 취향 기반 상호작용 부족  
- **가치 제안:** 리뷰·평점·댓글·좋아요·컬렉션을 한곳에서 제공하는 **영화 특화 커뮤니티**  

---

## 2) 목표 & 성공 지표 (North Star & KPI)
- **North Star:** *활성 컬렉션 생성 사용자 수*  
- **KPI(초기 가설/6주):**
  - 신규 가입 → 컬렉션 1건 이상 생성 전환율 **≥ 35%**
  - 리뷰 작성 유저 비율 **≥ 40%**
  - 7일 재방문율 **≥ 25%**
  - 검색 → 상세보기 전환율 **≥ 55%**

> 릴리스 1회차 이후 A/B 없는 단일 릴리스 기준으로 **피드백 기반 UX 개선** 후 지표 추적

---

## 3) 범위 정의 (MVP Scope)
- **핵심 기능:** TMDB 검색/상세, 리뷰/평점, 댓글/좋아요, 북마크(컬렉션), 인증/인가(JWT/OAuth2)  
- **비범위(De-scope):** 실시간 스트리밍, 추천 알고리즘 고도화, 외부 소셜 연동 자동 포스팅(차기 라운드)

---

## 4) 우선순위 (RICE Framework 요약)
| 기능 | Reach | Impact | Confidence | Effort | RICE |
|---|---:|---:|---:|---:|---:|
| 검색/상세 | 100 | 3 | 0.8 | 2 | 120 |
| 리뷰/평점 | 80 | 3 | 0.7 | 2 | 84 |
| 컬렉션(북마크) | 70 | 2 | 0.7 | 1 | 98 |
| 댓글/좋아요 | 70 | 2 | 0.7 | 2 | 49 |
| 소셜 공유 | 60 | 1 | 0.6 | 1 | 36 |

> **결정:** 검색/상세 → 컬렉션/리뷰 → 댓글/좋아요 → 공유 순

---

## 5) 일정 & 마일스톤 (6주 로드맵)
- **W1** 문제정의/목표·KPI/PRD v1, 위험 식별, 팀 RACI  
- **W2** UX 와이어프레임/플로우, API 1차 명세, 데이터 모델 초안  
- **W3** ERD 확정, 백엔드 스켈레톤, 프론트 구조 확정  
- **W4** 핵심 기능(검색/상세/컬렉션/리뷰) 구현, 1차 통합  
- **W5** 고도화(댓글/좋아요), QA 시나리오, 버그 픽스, 문서화  
- **W6** 베타 배포, 사용자(테스트 그룹) 피드백 수집, UX 개선 릴리스

> **게이트 리뷰:** W3, W5, W6에 **튜터/팀 중간점검**(스코프·리스크·버그 버닝다운)

---

## 6) 협업 거버넌스 (Agile/Scrum)
- **의사소통:** Slack(일일), Notion(요구/회의록/결정), GitHub Issues/Kanban(진행률)  
- **의식(Ceremonies):** 데일리 15’, 주간 스프린트 계획/회고, 중간점검(튜터/팀)  
- **품질:** QA 시나리오/체크리스트, Swagger 계약기반 협업, PR 템플릿/리뷰 규칙

## 8) 리스크 레지스터 (Top Risks & Mitigation)
- **API 제한/변경:** TMDB Rate limit → 캐싱/지수적 백오프, 장애 시 대체 플로우  
- **일정 지연:** 스코프 크립 → **MVP 고정, De-scope 원칙, 주간 게이트 리뷰**  
- **품질 저하:** 병목/버그 → QA 시나리오 선반영, PR 리뷰, 스모크 테스트  
- **데이터 일관성:** 리뷰·댓글 참조 무결성 → 트랜잭션 규칙/삭제 정책 명문화

## 10) 결과 & 인사이트 (Outcome over Output)
- **MVP 6주 내 배포 달성**(기획→개발→릴리스)  
- **UX 개선 임팩트:** 검색→상세 전환율/리뷰 작성 완료율 **향상**(초기 가설 대비 긍정 추세)  
- **프로세스 성과:** GitHub 기반 칸반/PR 규율 정착 → **협업 리드타임 단축**
