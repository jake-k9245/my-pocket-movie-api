# 🎬 My Pocket Movie (PM 포트폴리오)

> **“영화 애호가들을 위한 전용 커뮤니티 MVP”**  
> 6주간 **문제 정의 → 기획 → 설계 → 실행 → 피드백 개선** 전 과정을 PM으로 주도했습니다.  

---

## 📌 프로젝트 개요
- **문제 정의**  
  - OTT(넷플릭스·왓챠 등)는 시청 기록·추천 중심이라 **개인 맞춤형 컬렉션 관리·공유 기능이 부족**  
  - 인스타그램·유튜브 같은 범용 SNS는 **영화 전용 커뮤니티가 아니라 깊이 있는 취향 교류가 어려움**  
  - 기존 영화 리뷰 사이트는 일방향 정보 제공 위주로 **상호작용(댓글/좋아요/공유) 부족**  
  - → **개인화된 기록 관리 + 커뮤니티 상호작용**을 동시에 제공하는 서비스 부재

- **프로젝트 목적**  
  1) **개인화**: 찜·리뷰·평점을 컬렉션 형태로 관리  
  2) **소셜 상호작용**: 댓글/좋아요/공유로 커뮤니티 강화  
  3) **UX 개선**: 검색→상세→리뷰 작성 플로우 단순화  
  4) **검증 목표**: 6주 내 MVP 배포, 초기 사용자 피드백으로 핵심 가치 검증

- **기간**: 2025.05 ~ 2025.07 (6주)  
- **팀 규모**: 5명 (PM 1, 백엔드 3, 프론트엔드 1)  
- **성과 요약**: **6주 내 MVP 기획·개발·배포 완료, 사용자 피드백 반영 → 리뷰 UI·검색 정확도 개선**

---

## 🚀 내 기여 (PM Role)

### 1) 요구사항 정의
- 사용자 여정 기반 **검색→상세→리뷰→컬렉션→공유** 플로우 설계  
- **PRD/Acceptance Criteria**로 기준 통일 → 개발·QA 정렬  
- **효과**: 스코프 크립 예방, MVP 범위 합의

### 2) RICE 우선순위
- 기능을 Reach/Impact/Confidence/Effort로 점수화  
- **우선순위**: 검색/상세 → 리뷰/컬렉션 → 댓글/좋아요 → 공유  
- **효과**: 핵심 집중으로 6주 내 배포 달성

### 3) 일정·리스크 관리
- **6주 로드맵**: 문제정의→UX→ERD→개발→QA/고도화→배포  
- 주간 스프린트·**중간점검 회의(튜터/팀)**로 진행·리스크 시각화  
- API rate limit/일정 지연 대비 **대응 플랜**(캐싱, De-scope) 준비  
- **효과**: 일정 지연 0건, 계획 대비 온타임 릴리스

## 🤝 협업 방식 (PM 주도)

- **Notion – Single Source of Truth**  
  - PRD, 회의록, API 명세, 일정표를 Notion에 일원화하여 **모든 팀원이 동일한 최신 정보** 참조  
  - 결정/변경 기록을 축적 → 온보딩 속도 및 의사결정 투명성 향상

- **Slack – 실시간 이슈 관리 체계**  
  - 개발 중 이슈를 채널에서 즉시 공유, **중요 결정은 Notion에 기록**해 누락 방지  
  - 알림/리마인드로 커뮤니케이션 공백 최소화

- **GitHub Issues + Kanban – 태스크 가시화 & 추적**  
  - 이슈 단위로 작업 분해, **Backlog → In Progress → Done** 흐름을 시각화  
  - Sprint 계획과 연계해 **진행률·잔여 태스크·리스크** 투명 관리

- **Sprint 운영 – Agile/Scrum 적용**  
  - 1주 스프린트로 목표 설정, **Daily/Review/Retrospective**를 정례화  
  - 스프린트 종료마다 산출물/리스크 점검 → **프로세스 개선 포인트 도출**

**협업 성과**
- 협업 체계 정착으로 **PR 리뷰 리드타임 30% 단축**  
- 실시간 공유로 **중복 작업·커뮤니케이션 손실 최소화**  
- Notion·Slack·GitHub·Sprint 프로세스 정착 → **한눈에 상태 파악 가능한 운영 체계 구축**

---

## 📊 프로젝트 성과
- **6주 내 MVP 기획·개발·배포 완료**, 초기 사용자 피드백 반영으로 **리뷰 UI·검색 기능 개선 → UX 향상**  
- **RICE 기반 범위 관리**로 핵심 기능 집중 → 일정 지연 없이 프로젝트 완료  
- **협업 프로세스 정착**(Notion·Slack·GitHub Issues)으로 팀 커뮤니케이션 비용 절감, **생산성 향상**  
- **API 명세·ERD 문서화**로 개발-기획 간 협업 비용 최소화, 개발 효율 증대  
- 초기 사용자 테스트로 **핵심 가치(컬렉션·리뷰) 검증**, 차기 버전 확장 근거 확보
