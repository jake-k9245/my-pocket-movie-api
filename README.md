# 🎬 My Pocket Movie (PM 포트폴리오)

> **“영화 애호가들을 위한 전용 커뮤니티 MVP”**  
> 6주 동안 **문제 정의 → 기획 → 설계 → 실행 → 피드백 개선** 전 과정을 PM으로 주도했습니다.  

---

## 📌 프로젝트 개요
- **목적**: 기존 SNS(인스타·페북·유튜브)에는 영화 중심 소통 플랫폼이 부재 → 영화 팬들을 위한 **리뷰·컬렉션 중심 커뮤니티** 기획  
- **기간**: 2025.05 ~ 2025.07 (6주)  
- **팀 규모**: 5명 (PM 1, 백엔드 3, 프론트엔드 1)  
- **성과 요약**: **6주 내 MVP 기획·개발·배포, 사용자 피드백 반영 → 리뷰 UI·검색 정확도 개선**

---

## 🚀 내 기여 (PM Role)
- **요구사항 정의 & 기능 기획**: 사용자 여정 설계, RICE Framework로 MVP 범위 확정, PRD 작성  
- **일정 & 리스크 관리**: 6주 로드맵, 스프린트 운영, 중간점검 회의 주도 → 일정 지연 리스크 최소화  
- **협업 조율**: Slack/Notion 기반 문서 관리, GitHub Issues + Kanban으로 태스크 관리, Figma·Swagger 활용한 협업  
- **피드백 개선**: 사용자 테스트 결과 기반 UI 개선, QA 시나리오 정의 및 기능 안정성 확보  

---

## 📊 프로젝트 성과
- **MVP를 기한 내 배포 성공** (6주)  
- 사용자 피드백 기반 UI/UX 개선, 검색 성능 최적화  
- 협업 프로세스 정착 → GitHub Issues & Kanban으로 팀 생산성 향상  
- 문서화(API 명세·ERD) → 개발-기획 협업 비용 절감  

---

## 🛠 PM Methods & Tools
- **방법론**: Agile/Scrum, RICE Framework, Sprint Planning, PRD 작성  
- **툴**: Slack, Notion, GitHub Issues, Figma, Swagger, Postman  

---

## 👥 팀 구성
- **PM/리드 (김진환)**: 문제 정의, 기획, 일정 관리, 협업 조율  
- **BE 팀원 3명**: 검색/리뷰/인증 도메인 개발 및 배포  
- **FE 팀원 1명**: 와이어프레임·UI 설계 및 구현  
- **튜터**: 기술 검토 및 중간점검 피드백  

---

## 🌱 성장 포인트
- **문제 정의 → MVP → 개선 사이클**을 실무적으로 경험  
- 데이터 기반 의사결정(RICE, KPI 설정)과 Agile 프로세스 운영 경험  
- 협업 툴과 문서화를 통한 **팀 생산성 극대화**  
- 사용자 중심 사고를 제품 개선에 적용
