# 🎬 My Pocket Movie (PM 포트폴리오)

> **“영화 애호가들을 위한 전용 커뮤니티 MVP”**  
> 6주간 **문제 정의 → 기획 → 설계 → 실행 → 피드백 개선** 전 과정을 PM으로 주도했습니다.  

---

## 📌 프로젝트 개요
- **목적**: 인스타그램, 유튜브 같은 범용 SNS는 있지만 **영화 전용 소통 플랫폼은 부재** → 영화 팬들이 리뷰·컬렉션 중심으로 교류할 수 있는 공간 기획  
- **기간**: 2025.05 ~ 2025.07 (6주)  
- **팀 규모**: 5명 (PM 1, 백엔드 3, 프론트엔드 1)  
- **성과 요약**: **6주 내 MVP 기획·개발·배포 완료, 사용자 피드백 반영 → 리뷰 UI·검색 정확도 개선**

---

## 🚀 내 기여 (PM Role)
- **요구사항 정의 & 기능 기획**: 사용자 여정 설계, 핵심 기능 도출, RICE Framework로 우선순위 산정 → MVP 범위 확정  
- **일정 & 리스크 관리**: 6주 로드맵 수립, 주간 스프린트 운영, **중간점검 회의 주도** → 일정 지연 리스크 최소화  
- **협업 조율**: Slack + Notion으로 일정/문서 관리, GitHub Issues & Kanban으로 태스크 진행률 관리, Figma/Swagger 활용한 협업  
- **피드백 개선**: 사용자 테스트(팀원 대상) 진행 → 리뷰 UI 및 검색 성능 개선, QA 시나리오 정의 및 안정성 확보  

---

## 📊 프로젝트 성과
- **MVP를 기한 내 배포 성공** (6주)  
- 사용자 피드백 반영 → UI/UX 개선 및 검색 성능 최적화  
- 협업 프로세스 정착 → GitHub Issues + Kanban으로 팀 생산성 향상  
- API 명세·ERD 문서화를 통해 **개발-기획 협업 비용 절감**  

---

## 🛠 PM Methods & Tools
- **방법론**: Agile/Scrum, RICE Framework, PRD 작성, Sprint Planning  
- **툴**: Slack, Notion, GitHub Issues, Figma, Swagger, Postman  

---

## 👥 팀 구성
- **PM/리드 (김진환)**: 문제 정의, 기획, 일정 관리, 협업 조율  
- **BE 팀원 3명**: 검색·리뷰·인증/인가 개발, 배포 관리  
- **FE 팀원 1명**: UI/UX 와이어프레임 및 구현  
- **튜터**: 중간점검·리스크 검토, 기술 피드백  

---

## 🌱 성장 포인트
- 단순 기능 구현이 아닌 **문제 정의 → MVP → 개선 사이클** 실무 경험  
- 데이터 기반 의사결정(RICE, KPI 설정)과 Agile 프로세스 운영 경험  
- 협업 툴 활용 및 문서화를 통한 **팀 생산성 극대화**  
- 사용자 중심 사고로 **제품 개선 성과 창출**
