# 🎬 My Pocket Movie

> **“내가 찜한 영화, 나만의 컬렉션으로 정리하는 영화 커뮤니티 플랫폼”**  
> 본 프로젝트에서 **PM 역할**을 수행하며 **문제 정의 → 기획 → 설계 → 실행 → 피드백 개선** 전 과정을 주도했습니다.

---

## 📌 프로젝트 개요

**My Pocket Movie**는 단순 영화 정보 제공을 넘어,  
사용자가 TMDB API를 통해 영화를 검색하고, 리뷰·평점·댓글·좋아요 등 **소셜 기능을 통해 개인화된 영화 컬렉션**을 구축할 수 있는 커뮤니티 플랫폼입니다.  

- **프로젝트 목적 & 문제 정의**  
  - 기존 인스타그램, 페이스북, 유튜브 등은 범용적인 SNS이지만, **영화 소통만을 위한 전문 커뮤니티 플랫폼은 부족**  
  - 영화 애호가들이 리뷰·추천·컬렉션을 중심으로 교류할 수 있는 **특화된 영화 커뮤니티 플랫폼**을 만들고자 기획  

- **기간**: 2025.05 ~ 2025.07 (6주)  
- **팀 구성**: 5명 (PM/기획 1, 백엔드 3, 프론트엔드 1)  
- **성과 요약**: **6주 내 MVP 기획·개발·배포, 사용자 피드백 반영 및 개선 완료**  

---

## 🚀 내 기여 (PM Role)

- **요구사항 정의 & 기능 기획**  
  - 사용자 시나리오(검색·리뷰·찜하기 등) 작성  
  - 기능 우선순위를 **RICE Framework**로 산정 → MVP 범위 확정  

- **일정 및 리스크 관리**  
  - **주차별 프로젝트 로드맵 수립**  
    1주차: 문제 정의  
    2주차: UI/UX 시각화  
    3주차: ERD 작성  
    4주차: 핵심 기능 제작  
    5주차: 기능 고도화  
    6주차: 최종 배포  
  - 프로젝트 스케줄 관리 담당, 튜터와의 중간점검 회의 진행 → 완성도 체크  

- **협업 조율**  
  - 팀원들과 **Slack + Notion**을 통한 업무 관리 및 문서 공유  
  - GitHub Issues + Kanban 보드 활용 → 태스크 진행률 시각화 및 관리  

- **피드백 기반 개선**  
  - 사용자(팀원) 피드백 반영 → 리뷰 UI 개선 & 검색 정확도 향상  
  - QA 시나리오 정의 및 실행, 기능 안정성 확보  

---

## 📊 프로젝트 성과

- **6주 내 MVP 배포 성공**  
- 사용자 피드백 기반 UI/UX 개선 및 검색 성능 최적화  
- GitHub 기반 협업 프로세스 최적화 → 팀 생산성 향상  
- 데이터 모델링 & API 문서화 → 개발-기획 커뮤니케이션 비용 절감
